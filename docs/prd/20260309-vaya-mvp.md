## Phase 1: 아이디에이션 요약

### 핵심 유저 시나리오
개인 게임 유튜버가 OBS로 녹화한 게임 영상을 Vaya에 드롭하면, 자동으로 하이라이트 구간을 감지하고 자막을 생성하여, YouTube/Shorts용으로 내보내는 올인원 데스크톱 앱.

### 의사결정 기록 (Phase 1)

| 결정 사항 | 선택 | 근거 |
|-----------|------|------|
| 데스크톱 프레임워크 | Electrobun (Tauri v2 폴백) | 사용자 선호 + 경량화. Phase 1 3일 타임박스 |
| 백엔드 통신 | HTTP REST + WebSocket (경로 전달) | 바이너리 RPC 회피, 대용량 파일 안전 |
| 파일 업로드 방식 | 로컬 경로 전달 (복사 없음) | 10GB+ 파일 대응 |
| Python 실행 | uv run (자동 venv 관리) | 개발 편의성 |
| 인코딩 | NVENC (libx264 폴백) | GPU 가속 + 그레이스풀 디그레이드 |
| 세그먼트 병합 | 항상 재인코딩 (stream copy 금지) | 오디오 싱크 안정성 |

## Phase 2: 유저 시나리오 상세

### 시나리오 1: 앱 시작

**전제 조건**: Vaya 앱이 설치되어 있고, Python 백엔드 의존성이 설치됨
**단계**:
1. 사용자가 Vaya 앱을 실행한다
2. 화면 중앙에 **"VAYA"** 로고와 **"서버 시작 중..."** 로딩 스피너가 표시된다
3. Electrobun Main Process가 `uv run python backend/main.py`로 Python 자식 프로세스를 실행한다
4. Main Process가 `GET /api/health`를 500ms 간격으로 최대 10회 폴링한다
5. Health check 성공 시 → **드래그&드롭 화면**으로 전환
6. Health check 실패 시 → **에러 화면** 표시

**예외 처리**:
- 포트 8765 이미 사용 중: 기존 프로세스 확인 → health check 시도 → 성공하면 재사용, 실패하면 기존 프로세스 종료 후 재시작
- Python 미설치 / uv 미설치: 에러 화면에 "Python 환경을 확인해주세요" 메시지
- 에러 화면의 **[재시도]** 버튼: 로딩 화면으로 돌아가 프로세스 재시작

---

### 시나리오 2: 영상 업로드

**전제 조건**: 드래그&드롭 화면이 표시됨
**단계**:
1. 화면 중앙에 드래그&드롭 영역이 표시됨 (아이콘 + "영상 파일을 여기에 드래그하세요" + [파일 선택] 버튼)
2. 지원 포맷 안내: `MP4 · MKV · MOV · WEBM`
3. 사용자가 영상 파일을 드래그하거나 [파일 선택] 버튼 클릭
4. **[포맷 검증]** FFprobe로 파일 검증:
   - 성공 → 5단계로
   - 실패 → 토스트 알림: "지원하지 않는 형식: {파일명} ({코덱})" + 드래그&드롭 화면 유지
5. **[긴 영상 체크]** 영상이 1시간 이상이면:
   - 경고 모달: "이 영상은 {시간}입니다. 분석에 상당한 시간이 소요될 수 있습니다." + [취소] [계속 진행]
   - 취소 → 드래그&드롭 화면 유지
   - 계속 진행 → 6단계로
6. 웹뷰에서 파일 경로 추출 → RPC로 Main Process에 전달 → `POST /api/upload {"file_path": "..."}` 호출
7. 백엔드: 경로 검증 → UUID 할당 → FFprobe 메타데이터 추출 → 썸네일 생성 시작 (5초 간격)
8. **메인 화면으로 전환** (프로그레스 배너 포함)

**예외 처리**:
- 파일 경로에 한글/공백 포함: pathlib.Path 사용으로 안전 처리
- 파일 경로 미지원 (WebView2 제한): `openFileDialog()` 폴백

---

### 시나리오 3: 분석 (자동 시작)

**전제 조건**: 영상 업로드 완료, 메인 화면 표시
**단계**:
1. 업로드 즉시 분석이 **자동 시작**됨 (별도 버튼 없음)
2. 화면 상단에 **프로그레스 배너** 표시:
   - 프로그레스 바 + 퍼센트 + 현재 단계 텍스트
   - 단계: "오디오 추출 중..." → "오디오 분석 중..." → "영상 분석 중..." → "하이라이트 스코어링 중..." → "자막 생성 중..."
3. 동시에 **Whisper STT도 자동 실행** (분석과 병렬이 아닌 파이프라인 순서)
4. **메인 화면 레이아웃** (분석 중에도 표시):
   - **좌측 패널**: 영상 정보 (파일명, 해상도, FPS, 길이, 크기) + 하이라이트 구간 리스트 (분석 완료 후 표시) + 자막 리스트 (생성 완료 후 표시)
   - **우측 패널**: 분석 설정 슬라이더 + 내보내기 옵션
   - **하단**: 타임라인 (썸네일 로딩 중 → 완료 시 5초 간격 이미지 썸네일)
5. 분석 완료 시:
   - 프로그레스 배너 사라짐
   - 타임라인에 **하이라이트 구간 오버레이** 표시 (주황색)
   - 좌측 패널에 하이라이트 구간 리스트 표시 (#1, #2, ...)
   - 좌측 패널에 자막 리스트 표시 (시간 + 텍스트, 확인만 가능)
   - 내보내기 버튼 활성화
6. **하이라이트 구간이 0개**인 경우: 전체 영상을 1개 구간으로 자동 설정

**예외 처리**:
- 분석 중 에러 (FFmpeg 크래시, GPU 메모리 부족): 토스트 알림 + [재시도] 버튼, UI 조작 가능 상태로 복귀
- Whisper 첫 실행 시 모델 다운로드: 프로그레스에 "Downloading medium model (1.5GB)..." 표시
- GPU 미사용 시: Whisper CPU 폴백 (느리지만 동작)

---

### 시나리오 4: 하이라이트 편집

**전제 조건**: 분석 완료, 타임라인에 하이라이트 구간 표시됨
**단계**:
1. **타임라인 조작**:
   - 구간 양끝 드래그: 시작/끝 시간 조절 (비주얼 피드백)
   - 빈 영역 클릭: 새 구간 추가 (기본 5초)
   - 구간 선택 후 Delete 키 또는 우클릭: 구간 삭제
   - Ctrl+Z: Undo / Ctrl+Y: Redo
2. **좌측 패널 리스트 조작**:
   - 구간 리스트에서 시작/끝 시간 확인
   - [+ 구간 추가] 버튼
   - 각 구간 hover 시 [×] 삭제 버튼 표시
   - 수동 추가 구간은 파란색 표시 (자동 = 주황색, 수동 = 파란색)
3. **설정 변경 시 실시간 재계산**:
   - 슬라이더 조작 즉시 하이라이트 구간 재계산
   - **수동으로 추가/수정한 구간은 유지** (자동 구간만 재계산)
   - 토스트 알림: "수동 구간은 유지됩니다. 자동 구간만 재계산됩니다."
4. 모든 변경은 debounce 500ms 후 `PUT /api/video/{id}/highlights` 저장

**설정 패널 컨트롤**:

| 컨트롤 | 타입 | 범위 | 기본값 |
|---------|------|------|--------|
| 오디오 가중치 | Slider | 0-1, step 0.1 | 0.6 |
| 영상 가중치 | Display (1-오디오) | 자동 | 0.4 |
| 상위 구간 | Slider | 1-100% | 30% |
| 최소 클립 길이 | Number | 1-30s | 3s |
| 최대 클립 길이 | Number | 10-300s | 60s |
| 병합 간격 | Number | 0-10s | 2s |
| 무음 기준 | Slider | -60~-20dB | -40dB |
| 무음 최소 길이 | Number | 0.5-5s | 1.5s |
| Whisper 모델 | Select | tiny~large | medium |

### 시나리오 5: 내보내기

**전제 조건**: 하이라이트 구간이 1개 이상 존재
**단계**:
1. 우측 패널 **내보내기 섹션**:
   - 출력 폴더: `~/Videos/Vaya/` (기본) + [변경] 버튼
   - ☑ YouTube (16:9) — 1080p 60fps
   - ☑ Shorts (9:16) — 1080×1920
   - Shorts 체크 시 **크롭 오프셋 슬라이더** 표시 (← 좌 / 중앙 / 우 →)
   - ☑ 자막 포함
   - **[내보내기]** 버튼
2. [내보내기] 클릭 시:
   - 메인 화면 유지, 상단에 **프로그레스 배너** 표시
   - **전체 UI 비활성화** (수정 불가)
   - 프로그레스: "1/2 · YouTube용 인코딩 중..." → "2/2 · Shorts용 인코딩 중..."
   - YouTube 완료 후 Shorts **순차 실행**
3. 내보내기 완료 시:
   - 중앙에 **완료 카드** 표시:
     - ✓ 아이콘
     - 파일 리스트 (파일명 + 용량)
     - 출력 경로
     - **[폴더 열기]** 버튼
4. [폴더 열기] 클릭 → OS 파일 탐색기 오픈 → **드래그&드롭 초기 화면으로 자동 복귀**

**파일 이름 규칙**:
- `{원본파일명}_{날짜}_{포맷}.mp4`
- 예: `gameplay_20260309_youtube.mp4`, `gameplay_20260309_shorts.mp4`
- 동일 파일 존재 시: `gameplay_20260309_youtube(2).mp4` 번호 붙이기

**내보내기 사양**:

| 포맷 | 해상도 | FPS | 코덱 | 비트레이트 | 크롭 |
|------|--------|-----|------|-----------|------|
| YouTube 16:9 | 1920×1080 | 60 | h264_nvenc | 8Mbps | 없음 |
| Shorts 9:16 | 1080×1920 | 60 | h264_nvenc | 6Mbps | 중앙 + 사용자 오프셋 |

**무음 제거**: 하이라이트 구간 내부에서만 무음 구간 제거 (하이라이트 밖은 이미 잘려나감)

**예외 처리**:
- NVENC 미지원: libx264 폴백 + 경고 토스트
- 디스크 공간 부족: 에러 토스트 + UI 복귀
- FFmpeg 크래시: 에러 토스트 + [재시도] 버튼
- 인코딩 중 앱 종료: 다음 실행 시 미완성 파일 정리 (MVP에서는 수동)

## Phase 3: 검증 및 개발 계획

### 검증 환경
- **도구**: Playwright (브라우저 자동화 E2E)
- **백엔드 테스트**: pytest + httpx (API 단위 테스트)
- **환경**: Electrobun 웹뷰 내부를 Playwright로 제어, 또는 Python 백엔드를 독립 실행 후 API 검증

### 검증 시나리오

#### 정상 시나리오

| # | 시나리오 | 기대 결과 |
|---|---------|-----------|
| 1 | 앱 실행 → 로딩 화면 → 드래그&드롭 화면 | 로딩 스피너 표시 후 드래그&드롭 영역 활성화, health check 통과 |
| 2 | MP4 파일 업로드 (파일 선택 버튼) | 메인 화면 전환, 메타데이터 표시 (해상도, FPS, 길이, 크기) |
| 3 | 업로드 후 자동 분석 시작 | 프로그레스 배너 표시, 단계별 텍스트 변경, 퍼센트 증가 |
| 4 | 분석 완료 → 하이라이트 표시 | 타임라인에 주황색 오버레이, 좌측 패널에 구간 리스트, 자막 리스트 |
| 5 | 타임라인에서 구간 양끝 드래그 | 구간 시작/끝 시간 변경, 리스트 동기화 |

- [ ] [폴더 열기] → OS 파일 탐색기 → 드래그&드롭 초기 화면 복귀

**완료 조건**:
- 내보내기 → 프로그레스 표시 → 완료 카드
- UI가 내보내기 중 비활성화
- 폴더 열기 후 초기 화면 복귀
